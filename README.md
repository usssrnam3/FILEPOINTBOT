# FilePoint Telegram Bot MVP

MVP Telegram-бота для продажи цифровых файлов, гайдов и услуг. Проект рассчитан на одного разработчика и дешевый VPS: один контейнер с ботом, PostgreSQL, Redis, локальное приватное файловое хранилище без публичной раздачи.

## Что входит в MVP

- просмотр каталога в Telegram
- покупка цифрового товара
- автоматическая выдача файла после успешной оплаты
- покупка услуги с созданием оплаченного заказа
- админское создание товаров и публикация/снятие с публикации
- хранение файлов вне публичного доступа, выдача только ботом
- абстракция платежного провайдера с `demo`-реализацией

## Что не входит в MVP

- интеграция с реальным эквайрингом
- возвраты и спорные платежи
- промокоды, подписки, корзина
- CRM, аналитика, рассылки
- роли админов и полноценный backoffice
- CDN/object storage, шифрование файлов на диске, KMS

## Платежи в текущей версии

Сейчас подключен только `DemoPaymentProvider`: он имитирует успешную оплату кнопкой внутри Telegram и нужен, чтобы можно было поднять проект и проверить полный жизненный цикл покупки.

Заглушками являются:

- подтверждение оплаты без реального списания
- отсутствие webhook/callback от внешнего платежного сервиса
- отсутствие фискализации и статусов возврата

## Архитектура

- `aiogram 3` как Telegram-интерфейс и точка входа для всех сценариев
- модульный монолит без избыточных микросервисов
- `PostgreSQL` для транзакционных данных
- `Redis` для FSM и идемпотентной выдачи файлов
- `SQLAlchemy async` для доступа к БД
- `Alembic` для миграций
- локальное файловое хранилище в `storage/files`

## Ключевые сущности

- `users`: покупатели и админы в разрезе Telegram ID
- `products`: товары и услуги
- `product_files`: привязка цифрового файла к товару
- `orders`: заказы на товар или услугу
- `payments`: экземпляры платежей, связанные с заказами

## Безопасность MVP

- доступ в админку только по `ADMIN_IDS`
- файлы не отдаются по прямым ссылкам
- выдача файла идет только из бота после оплаченного заказа
- для авто-выдачи используется Redis-lock, чтобы уменьшить риск двойной отправки
- повторная выдача файла доступна только владельцу заказа

## Быстрый запуск

1. Скопируйте `.env.example` в `.env` и заполните значения.
2. Запустите `docker compose up --build`.
3. После старта напишите боту `/start`.
4. Администратор может открыть `/admin` и создать товары.

## Перед реальным запуском улучшить

- заменить `demo`-провайдер на YooKassa, Telegram Payments или другой эквайринг
- добавить webhook-процессинг и проверку подписи провайдера
- включить журнал аудита действий админа
- вынести файлы в S3-совместимое хранилище или добавить шифрование на диске
- ограничить размер и типы загружаемых файлов
- добавить мониторинг, резервное копирование и обработку ошибок доставки
